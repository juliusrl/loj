# Loj プロジェクトファイル仕様 (loj.project.yaml)

このドキュメントでは、Loj フルスタックアプリケーションのオーケストレーション層として機能する `loj.project.yaml` ファイルのスキーマと動作について定義します。

## 目的

`loj.project.yaml` は以下の責任を持ちます：
- アプリケーションを構成するターゲット（フロントエンド、バックエンドなど）の定義
- 各ターゲットの DSL ソースコードのエントリポイントの指定
- 生成されたコードの出力ディレクトリの設定
- 開発環境（デ部ホストおよびサーバー）のオーケストレーション
- 生成されたターゲットに対するデータベースおよびランタイム プロファイルの提供

このファイルには、ビジネスセマンティクス（モデル、リソースなど）を**含めてはなりません**。これらは `.web.loj` または `.api.loj` ファイルに記述します。

---

## 最小構成例

```yaml
app:
  name: user-admin

targets:
  frontend:
    type: web
    entry: frontend/app.web.loj
  backend:
    type: api
    entry: backend/app.api.loj
```

## フィールドリファレンス

### トップレベルセクション

| フィールド | 状態 | 説明 |
| --- | --- | --- |
| `app` | **必須** | アプリケーションの一般的なメタデータ。 |
| `targets` | **必須** | 論理ターゲットエイリアスとその設定のマップ。 |
| `dev` | オプション | 開発時のオーケストレーション設定。 |

### `app` セクション

| フィールド | 状態 | 説明 |
| --- | --- | --- |
| `name` | **必須** | アプリケーションの論理名（例：`invoice-system`）。 |

### `targets` セクション

`targets` 内の各エントリは、キーが**論理エイリアス**（例：`frontend`, `backend`, `admin-api`）のマップです。

| フィールド | 状態 | 説明 |
| --- | --- | --- |
| `type` | **必須** | `web` (フロントエンド) または `api` (バックエンド)。 |
| `entry` | **必須** | 项目ファイルからの相対パスとしての `.web.loj` または `.api.loj` のエントリファイル。 |
| `outDir` | オプション | 生成されたコードが配置されるディレクトリ。省略した場合はターゲット固有の慣習に従います。 |
| `database` | オプション | データベース設定（主に `api` ターゲット用）。 |
| `runtime` | オプション | ランタイム環境設定（ベースパス、CORS など）。 |

#### `database` スライス

`api` ターゲットでサポートされます。

| フィールド | 状態 | 説明 |
| --- | --- | --- |
| `vendor` | **必須** | `h2`, `sqlite`, `postgres`, `mysql`, `mariadb`, `sqlserver`, `oracle`。 |
| `mode` | オプション | `embedded` (埋め込み), `external` (外部), または `docker-compose`。 |
| `name` | オプション | データベース名。デフォルトは `app.name` から派生した名前です。 |
| `host` | オプション | データベースホスト。外部ベンダの場合はデフォルトで `127.0.0.1`。 |
| `port` | オプション | データベースポート。 |
| `username` | オプション | データベースユーザー名。 |
| `password` | オプション | データベースパスワード。 |
| `autoProvision`| オプション | `true` の場合、プロジェクトシェルが自動的にデータベースを開始/プロビジョニングすることがあります（例：Docker 経由）。 |
| `migrations` | オプション | `none`, `native-sql`, または `flyway`。 |

#### `runtime` スライス

| フィールド | 状態 | 説明 |
| --- | --- | --- |
| `basePath` | オプション | サービスの URL ベースパス（`api` および `web` でサポート）。 |
| `shutdown` | オプション | シャットダウン動作（`api` のみ）。`mode` (`graceful`\|`immediate`) と `timeout` を含みます。 |
| `health` | オプション | ヘルスチェックエンドポイントのパス（例：`/health`）。 |
| `readiness` | オプション | レディネスプローブエンドポイントのパス（例：`/ready`）。 |
| `cors` | オプション | CORS 設定（`api` のみ）。`origins` が必須です。 |
| `forwardedHeaders` | オプション | 転送ヘッダーモード (`standard`\|`none`)。 |
| `trustedProxy` | オプション | 信頼されたプロキシ設定。 |
| `requestSizeLimit` | オプション | 最大リクエストボディサイズ（例：`10mb`）。 |

---

### `dev` セクション (開発オーケストレーション)

`loj dev` によって開始される管理型開発ループを設定します。

#### `dev.host` (フロントエンドホスト)

| フィールド | 状態 | 説明 |
| --- | --- | --- |
| `target` | **必須** | `web` ターゲットのエイリアス。 |
| `dir` | **必須** | ホストプロジェクト（Vite/React プロジェクトなど）を含むディレクトリへのパス。 |
| `type` | オプション | `react-vite` (デフォルト)。 |
| `port` | オプション | 開発サーバーのポート（デフォルト `5173`）。 |
| `apiBase` | オプション | プロキシプレフィックス（デフォルト `/api`）。 |
| `proxyTarget`| オプション | プロキシ先の `api` ターゲットのエイリアス。 |

#### `dev.server` (バックエンドサーバー)

| フィールド | 状態 | 説明 |
| --- | --- | --- |
| `target` | **必須** | `api` ターゲットのエイリアス。 |
| `port` | オプション | バックエンドサーバーのポート（デフォルト `3001`）。 |

## 環境変数 (Environment Variables)

プロジェクトシェルは、以下のファイルから環境変数を自動的にロードします（プロジェクトファイルからの相対パス）：
1.  `.env`
2.  `.env.local`
3.  `.env.<target-alias>`
4.  `.env.<target-alias>.local`

`.env.local` は `.env` を上書きします。ターゲット固有のファイルは、一般的なファイルよりも優先されます。

---

## ガードレール

- **オーケストレーション専用**: ここにロジックやビジネスルールを追加しないでください。
- **設定より規約 (Convention over Configuration)**: デフォルトがターゲットフレームワークのベストプラクティスに適合しない場合にのみ、オーバーライドを使用してください。
- **相対パス**: `loj.project.yaml` 内のすべてのパスは、そのファイルが存在するディレクトリからの相対パスです。
